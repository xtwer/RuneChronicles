# 测试执行指南

## 自动化测试

### 1. 运行时测试（RuntimeTests）

**设置步骤：**
1. 打开 Unity Editor
2. 创建新的场景（或使用现有场景）
3. 在 Hierarchy 中创建空 GameObject，命名为 "TestRunner"
4. 将 `RuntimeTests.cs` 脚本挂载到 TestRunner 上
5. 在 Inspector 中勾选 "Auto Run On Start"
6. 运行场景（Play）

**查看结果：**
- 所有测试结果会输出到 Console
- ✓ PASS（绿色）= 测试通过
- ✗ FAIL（红色）= 测试失败
- 最终显示测试通过率

**配置选项：**
- `autoRunOnStart`: 启动时自动运行所有测试
- `pauseOnFailure`: 测试失败时暂停游戏（便于调试）

## 手动测试清单

### 游戏完整流程测试

**测试环境：** Unity Editor Play Mode

#### 1. 主菜单流程
- [ ] 启动游戏，显示主菜单
- [ ] 主菜单包含：开始游戏、设置、退出按钮
- [ ] 点击"设置"，可打开设置界面
- [ ] 设置界面可调整音量
- [ ] 关闭设置后返回主菜单

#### 2. 角色选择流程
- [ ] 点击"开始游戏"，进入角色选择
- [ ] 至少显示 2 个可选角色（法师/战士）
- [ ] 选择角色后进入地图

#### 3. 地图系统流程
- [ ] 地图显示 15 层节点
- [ ] 每层有 3 个可选节点
- [ ] 节点类型清晰（战斗/商店/宝箱/精英/Boss）
- [ ] 点击节点可进入对应事件

#### 4. 战斗系统流程
- [ ] 进入战斗，显示敌人和玩家
- [ ] 玩家回合开始：恢复 3 点能量，抽 5 张牌
- [ ] 可以打出手牌（消耗能量）
- [ ] 攻击牌造成伤害
- [ ] 技能牌产生效果（护盾/治疗等）
- [ ] 点击"结束回合"，进入敌人回合
- [ ] 敌人执行行动并造成伤害
- [ ] 击败所有敌人后显示奖励界面

#### 5. 奖励系统流程
- [ ] 战斗胜利后显示奖励选项
- [ ] 可选择卡牌/金币/融合点
- [ ] 选择后加入背包/库存
- [ ] 可跳过奖励并继续

#### 6. 商店系统流程
- [ ] 进入商店，显示商品列表
- [ ] 商品包含卡牌/遗物/消耗品
- [ ] 商品不重复
- [ ] 购买时检查金币是否足够
- [ ] 购买成功后扣除金币并获得物品
- [ ] 退出商店返回地图

#### 7. 融合系统流程
- [ ] 地图界面可打开融合界面
- [ ] 选择两张卡牌进行融合
- [ ] 如有预设配方，生成对应融合卡
- [ ] 如无配方，生成随机融合卡
- [ ] 融合消耗融合点
- [ ] 融合卡加入牌库

#### 8. 暂停与设置流程
- [ ] 游戏中按 ESC，打开暂停菜单
- [ ] 暂停菜单包含：继续/设置/返回主菜单/退出
- [ ] 点击"设置"，可调整音量
- [ ] 音量设置持久化保存
- [ ] 点击"继续"，返回游戏
- [ ] 点击"返回主菜单"，有确认弹窗

#### 9. 胜利与失败流程
- [ ] 击败第 15 层 Boss，显示胜利界面
- [ ] 胜利界面显示统计数据
- [ ] 玩家死亡，显示失败界面
- [ ] 失败界面可重新开始或返回主菜单
