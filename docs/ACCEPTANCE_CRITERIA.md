# 开发路线图验收标准补充 (ROADMAP - Acceptance Criteria)

> **本文档是 ROADMAP.md 的补充**，详细定义每个 Week 的验收标准

## Week 1-3 验收标准：核心原型

### Week 1：框架搭建

#### 功能验收
- [ ] Unity 项目创建成功（Unity 2022.3.15 LTS）
- [ ] Git 仓库初始化完成
- [ ] 核心管理器框架可运行：
  - [ ] GameManager 单例可用
  - [ ] BattleManager 可创建
  - [ ] CardManager 可创建
  - [ ] EnemyManager 可创建
  - [ ] FusionManager 可创建

#### 可测试性验收
- [ ] 实现至少 5 个 DEBUG 测试接口
- [ ] Debug Console 基础版完成（支持 hp, energy, add 命令）
- [ ] 编写至少 5 个单元测试（全部通过）

#### 技术验收
- [ ] 代码编译无错误
- [ ] 可在 Unity Editor 中运行
- [ ] 性能监控面板显示 FPS

**验收负责人：** 技术负责人  
**验收时间：** 周五下午

### Week 2：战斗系统

#### 功能验收
- [ ] 简单战斗流程可运行：
  - [ ] 玩家可打出卡牌
  - [ ] 敌人可受到伤害
  - [ ] 敌人可反击
  - [ ] 回合交替正常
- [ ] 10 张卡牌可用（ATK_001 ~ ATK_010）
- [ ] 3 种敌人可用（符文傀儡、灵狐、岩石巨人）

#### 数据验收
- [ ] 卡牌数据从 JSON 加载
- [ ] 敌人数据从 JSON 加载
- [ ] 数据校验脚本通过（validate_cards.py）

#### 性能验证
- [ ] 战斗帧率 ≥30 FPS
- [ ] 无明显卡顿
- [ ] 内存占用 <300 MB

#### 测试验收
- [ ] 至少 10 个单元测试（全部通过）
- [ ] 至少 3 个集成测试（全部通过）
- [ ] 手动完成 5 场战斗，无崩溃

**验收负责人：** 测试负责人  
**验收时间：** 周五下午

---

### Week 3：融合系统 + 原型验证 ⭐ **关键里程碑**

#### 功能验收
- [ ] 融合系统可运行：
  - [ ] 可选择 2 张卡进行融合
  - [ ] 融合消耗 3 融合点
  - [ ] 生成的融合卡正确（名称、效果、数值）
- [ ] 至少 5 种预设融合配方工作正常
- [ ] 随机融合算法工作正常

#### 性能验收（必须通过）
- [ ] **1000 次融合 <1 秒**（压力测试）
- [ ] 单次融合 <10ms
- [ ] 融合动画播放流畅（≥30 FPS）

#### 玩法验收（关键验证）
**邀请 5 名测试玩家，每人试玩 30 分钟**

| 验收项 | 目标 | 实际 | 状态 |
|-------|------|------|------|
| **融合系统趣味性** | ≥7 分（1-10分）| ___ | ⏳ |
| **融合系统易理解性** | 3 分钟内理解 | ___ | ⏳ |
| **融合使用频率** | 每局 ≥3 次 | ___ | ⏳ |
| **战斗节奏** | 单局 5-8 分钟 | ___ | ⏳ |
| **整体趣味性** | ≥6 分 | ___ | ⏳ |

**关键决策点：**
- ✅ **通过（≥7分）** → 继续 Week 4
- ⚠️ **及格（6-7分）** → 小幅调整后继续
- ❌ **不通过（<6分）** → 重新设计融合系统（延期 2 周）

#### 技术验收
- [ ] 单元测试覆盖率 ≥60%
- [ ] 无 P0/P1 缺陷
- [ ] 战斗重放系统可用（可保存/加载/重放）

**验收负责人：** 游戏设计师 + 测试负责人  
**验收时间：** 周五全天  
**风险等级：** 🔴 极高（原型失败则项目重大调整）

## Week 4-7 验收标准：内容扩充

### Week 4：全卡牌实现

#### 功能验收
- [ ] 80 张卡牌全部实现（ATK_001 ~ DEF_015 ~ ABL_015）
- [ ] 所有卡牌效果正确：
  - [ ] 攻击卡（40张）：伤害计算正确
  - [ ] 技能卡（25张）：防御/抽牌/回血/特殊效果正确

- [ ] 所有卡牌通过数据校验脚本

#### 测试验收
- [ ] 每张卡至少有 1 个单元测试
- [ ] 单元测试覆盖率 ≥70%
- [ ] 手动测试至少 30 张卡，无明显 bug

**验收负责人：** 功能开发 + 测试  
**验收时间：** 周五下午

### Week 7：平衡性测试 ⭐ **关键里程碑**

#### 自动化测试验收
**运行 100 局模拟游戏（难度 A5）**

| 指标 | 目标 | 实际 | 状态 |
|------|------|------|------|
| **胜率** | 40%-60% | ___ | ⏳ |
| **平均游戏时长** | 30-40 分钟 | ___ | ⏳ |
| **融合使用率** | ≥30% | ___ | ⏳ |

#### 卡牌使用率分析
- [ ] 无卡牌使用率 <1%（太弱）
- [ ] 无卡牌使用率 >50%（太强）
- [ ] TOP 10 卡牌使用率总和 <70%（避免"必选卡"）

#### 玩家反馈验收
**邀请 10 名测试玩家，每人玩 3 局（约 2 小时）**

| 反馈项 | 目标 | 实际 | 状态 |
|-------|------|------|------|
| **游戏难度** | 6-8 分（适中）| ___ | ⏳ |
| **卡牌平衡性** | ≥7 分 | ___ | ⏳ |
| **融合系统趣味性** | ≥7 分 | ___ | ⏳ |
| **整体推荐度** | ≥7 分 | ___ | ⏳ |

**关键决策点：**
- ✅ **通过** → 继续 Week 8
- ⚠️ **部分问题** → 调整数值后重新测试（延期 3-5 天）
- ❌ **严重失衡** → 大幅调整数值（延期 1 周）

**验收负责人：** 数值策划 + 测试负责人  
**验收时间：** 周五全天 + 周末  
**风险等级：** 🔴 高（平衡性影响游戏体验）

## 📊 总体质量门槛

**只有满足以下所有条件，才能发布 Early Access：**

| 门槛项 | 标准 | 检查人 |
|-------|------|--------|
| ✅ 功能完整性 | 核心玩法可玩，无阻塞性 bug | 测试负责人 |

| ✅ 无致命缺陷 | 无 P0/P1 缺陷 | 测试负责人 |

**任何一项不通过 → 延期发布，直到通过为止**
